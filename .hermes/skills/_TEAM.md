# Équipe IA — Business Ascension™

> L'équipe d'agents de Christofer. Chaque **rôle** (département) contient des **sous-agents** (employés spécialisés).
> Chaque sous-agent est un fichier `.md` = un prompt système prêt à l'emploi.

## Comment s'en servir
- **Dans Hermes Desktop / Telegram** : « Prends le rôle Hook Generator » + ta demande. (Hermes lit le contexte BA via son `SOUL.md` + ces skills.)
- **Dans Claude Code (ici, à ton bureau)** : ouvre le fichier du sous-agent, colle-le, ou réfère-toi à lui. Meilleur modèle = meilleure sortie.
- **Source de vérité commune** : tous lisent d'abord `01-MARKETING/_fondations/` (ma-voix, ma-cible, mes-offres, voc-langage-client).

## Règles communes à TOUS les agents
- Français, tutoiement, oral, direct. Zéro tiret cadratin.
- Voix de Chris + VOC (vivoter, goulot, plafond de verre…). Mots interdits respectés (liberté financière, revenus passifs, etc.).
- Branding : identité d'abord. Hook = croyance, jamais la douleur étalée.
- Chaque sortie sert UN but : produire, vendre, ou closer. Sinon le dire.

## Pourquoi cet ordre
La précision d'un agent vient des **vrais retours clients**, pas du nombre d'agents. On construit d'abord ce qui amène des clients, on nourrit la mémoire avec leurs retours réels, puis on étend. Voir la règle anti-build-trap de Chris.

## Mémoire par agent
Chaque sous-agent peut accumuler ses apprentissages dans `.hermes/memory/<role>-<agent>.md` (ex : ce qui a converti, les hooks qui ont marché). À nourrir avec du RÉEL (posts qui ont performé, DM qui ont closé), pas du théorique.
